## GitHub Pages部署

本项目已完全适配GitHub Pages，支持两种部署方式：

### 🚀 快速部署（推荐）

1. **创建GitHub仓库**：在GitHub上创建新仓库（建议名称：`DigitalLawBriefing202503`）
2. **上传文件**：将本文件夹内所有文件上传到仓库
3. **启用Pages**：
   - 进入仓库的 **Settings → Pages**
   - Source 选择 **"Deploy from a branch"**
   - Branch 选择 **"main"** 或 **"master"**
   - Folder 选择 **"/ (root)"**
   - 点击 **Save**
4. **访问网站**：等待部署完成后访问 `https://[username].github.io/DigitalLawBriefing202503/`

### ⚡ 自动部署（GitHub Actions）

项目已配置GitHub Actions，支持自动部署：

1. **上传文件**：将所有文件推送到GitHub仓库
2. **自动触发**：GitHub Actions会自动运行部署流程
3. **查看状态**：在 **Actions** 标签页查看部署进度
4. **访问网站**：部署完成后访问 `https://[username].github.io/DigitalLawBriefing202503/`

### 📁 关键部署文件

- `.nojekyll`：告诉GitHub Pages跳过Jekyll处理
- `_config.yml`：Jekyll配置文件
- `.github/workflows/deploy.yml`：GitHub Actions自动部署配置
- `404.html`：自定义404错误页面
- `DEPLOY.md`：详细部署指南

### 🔧 部署要求

- **GitHub账户**：需要有效的GitHub账户
- **仓库权限**：确保仓库启用了Pages功能
- **文件大小**：单个文件建议不超过100MB
- **公共仓库**：GitHub Pages免费版需要公共仓库

## 🔧 故障排除

### 常见问题

**Q: GitHub Pages部署失败**
A: 检查仓库设置中的Pages配置，确保选择了正确的分支和文件夹

**Q: 页面样式显示异常**
A: 确保CSS文件路径正确，检查是否有语法错误

**Q: 搜索功能不工作**
A: 确保JavaScript文件正确加载，检查浏览器控制台错误信息

**Q: 大文件无法上传**
A: GitHub单文件限制为100MB，超过大小需要使用Git LFS或分割文件

**Q: 中文文件名404错误**
A: GitHub Pages可能对中文文件名进行URL编码，建议使用英文文件名

### 性能优化建议

- 使用相对路径引用文件
- 压缩图片和PDF文件
- 启用GitHub Pages的缓存
- 定期清理不需要的文件
